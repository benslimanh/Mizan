Security

Last updated: 2026

Blockchain-Powered, Sharia-Compliant (AAOIFI standards) controls built on an Immutable Ledger—delivering Ethical Transparency with Smart Contracts for Murabaha/Sukuk.

Security Principles

  • Least privilege by default for all access.
  • Auditability of workflow actions and administrative changes.
  • Clear, immutable contract terms to reduce Gharar (uncertainty).
  • Defense-in-depth across application, network, and data layers.

Sharia Integrity

For Sharia-compliant transactions, clarity is security. Blockchain-style append-only records and structured contract fields help prevent Gharar by keeping terms explicit, consistent, and reviewable across approvals and audits.

Data Protection

  • Encryption in transit using modern TLS configurations.
  • Encryption at rest where applicable across storage layers.
  • Secure key management and controlled access to secrets.

Access Controls

Role-based permissions, strong authentication options, and clear administrative audit logs support Sharia governance, reviews, and On-chain AAOIFI Standards Compliance.

Vulnerability Reporting

If you believe you’ve found a security issue, please report it through the Contact page. Include steps to reproduce and any relevant details.
